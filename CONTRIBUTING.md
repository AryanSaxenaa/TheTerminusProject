# Contributing to The Terminus Project

We welcome contributions to The Terminus Project! Whether you're fixing bugs, adding features, or improving documentation, your help is appreciated.

## How to Contribute

1.  **Fork the Repository**: Create your own copy of the project.
2.  **Create a Branch**: Make your changes in a new branch (e.g., `feature/awesome-feature` or `fix/critical-bug`).
3.  **Commit Changes**: Write clear, concise commit messages.
4.  **Push to GitHub**: Upload your branch to your forked repository.
5.  **Open a Pull Request**: Submit your changes for review.

## Coding Standards

*   **Tech Stack**: We use **Vanilla JavaScript, HTML, and CSS**. No frameworks (React, Vue, etc.) or build tools (Webpack, Vite) are allowed unless discussed first.
*   **Style**:
    *   Keep code readable and commented where necessary.
    *   Follow the existing maximizing of ASCII art usage.
    *   Maintain the retro/sci-fi aesthetic in all UI additions.
*   **Performance**: Since this runs in the browser without a backend, ensure code is efficient and doesn't cause lag.

## Reporting Issues

If you find a bug or have a suggestion, please open an issue in the GitHub repository. providing:
*   Steps to reproduce the bug.
*   Expected vs. actual behavior.
*   Browser and OS details.

## Issue Assignment & Policy

*   **Assignment**: Issues are assigned on a **first-come, first-served** basis. Please comment on an issue to claim it.
*   **Anti-Spam**: Spamming issues or Pull Requests will not be tolerated.
*   **Timeouts**: To keep development moving, assigned issues will be reassigned if there is no activity or submission within:
    *   **Easy Issues**: 3 days
    *   **Medium Issues**: 5 days
    *   **Hard Issues**: 7 days

## Feature Requests

Check `IMPROVEMENT_SUGGESTIONS.md` for a list of planned improvements. If you want to work on one of these, please mention it in an issue first to avoid duplicate work.

Thank you for helping rebuild Sector-7!
